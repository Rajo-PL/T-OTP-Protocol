English Version:
Fiscal Printer Integration (Hardware Oracle Priority 1)
The core of the T-OTP protocol is the integration with a GUM-homologated fiscal printer.
* [cite_start]**Source of Trust:** The system bypasses the unreliable PC system clock and retrieves the timestamp directly from the printer's sealed Real-Time Clock (RTC) [cite: 12, 65-66, 143].
* [cite_start]**Fiscal Binding:** Each offline transaction is cryptographically "bound" to a unique fiscal event number[cite: 36, 167].
* [cite_start]**Hard Lock Mechanism:** The "Hard Lock Controller" ensures that no fiscal receipt is printed unless the T-OTP "Frozen Blob" has been successfully generated and stored[cite: 14, 145, 150].

Wersja Polska:
Integracja z drukarką fiskalną (Hardware Oracle Priorytet 1)
Sercem protokołu T-OTP jest integracja z drukarką fiskalną posiadającą homologację GUM.
* [cite_start]**Źródło zaufania:** System pomija niewiarygodny zegar systemowy komputera i pobiera znacznik czasu bezpośrednio z zaplombowanego modułu RTC drukarki [cite: 12, 65-66, 143].
* [cite_start]**Wiązanie fiskalne:** Każda transakcja offline jest kryptograficznie „wiązana” z unikalnym numerem zdarzenia fiskalnego[cite: 36, 167].
* [cite_start]**Mechanizm Hard Lock:** Kontroler „Hard Lock” gwarantuje, że żaden paragon fiskalny nie zostanie wydrukowany, dopóki pakiet „Frozen Blob” T-OTP nie zostanie poprawnie wygenerowany i zapisany[cite: 14, 145, 150].
